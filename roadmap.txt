- Shared: ai/, uploads/ (models separated by database)

## DETAILED BUSINESS WORKFLOW

### 1. Business Onboarding Process
```
Business Signup → Email Verification → WhatsApp API Setup → Agent Creation → Webhook Configuration
```

**Step-by-Step:**
1. **Signup**: Business creates account (name, email, password)
2. **WhatsApp Setup**: Business provides:
   - WhatsApp Phone Number ID
   - WhatsApp Access Token
   - Webhook Verify Token
   - App Secret
3. **Agent Creation**: Business creates AI agents with multilingual instructions
4. **Webhook Config**: Business sets webhook URL in Facebook Developer Console:
   - URL: `https://yourdomain.com/webhook`
   - Verify Token: Their unique token

### 2. Message Processing Flow
```
WhatsApp User → Facebook → Our Webhook → Business Lookup → Agent Processing → AI Response → WhatsApp User
```

**Detailed Flow:**
1. **Incoming Message**: Facebook sends webhook to `/webhook`
2. **Business Identification**: Extract `phone_number_id` from payload
3. **Database Lookup**: `Business.findOne({whatsapp_phone_id: phone_number_id})`
4. **Agent Context**: Load business-specific agent instructions
5. **AI Processing**: Process message with agent context
6. **Response**: Send back using business WhatsApp token

### 3. Multi-Tenant Isolation
- **Database Level**: Separate databases prevent data mixing
- **Credential Level**: Each business uses own WhatsApp tokens
- **Agent Level**: Business-specific AI instructions and context
- **Error Level**: Business errors don't affect others

## TECHNICAL DECISIONS

### Database Strategy
- **Existing Bot**: 'whatsapp_bot' database (unchanged)
- **SaaS Platform**: 'whatsapp_saas' database (new)
- Complete data isolation through separate databases
- Cross-database queries only when needed
- Business identification via whatsapp_phone_id

### Authentication
- JWT tokens for API authentication
- Session-based for web interface
- bcrypt for password hashing
